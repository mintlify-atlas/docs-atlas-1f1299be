The `iconOptions` prop gives you full control over the appearance of navigation buttons, including custom icons and styling.

## Basic Usage

The `iconOptions` prop accepts an object with two properties:

```tsx
import { Carousel } from 'react-any-carousal';

<Carousel 
  iconOptions={{
    icon: <span>→</span>,
    iconStyles: { color: 'blue', backgroundColor: 'white' }
  }}
>
  {items.map((item) => <div key={item.id}>{item.content}</div>)}
</Carousel>
```

## IconOptions Type

From the source code (`carousel-props.ts:19-22`):

```typescript
export type IconOptions = {
  icon?: ReactNode;
  iconStyles?: any;
};
```

<Note>
  The same icon is used for both left and right navigation buttons. The right button automatically mirrors the icon.
</Note>

## Default Icons

By default, AnyCarousel uses Unicode chevron characters:

```tsx
const defaultIconOptions = {
  icon: <span>&#8249;</span>, // Left single angle quotation mark (‹)
  iconStyles: { color: 'whitesmoke', backgroundColor: '#333' }
};
```

## Styling Icons

The `iconStyles` property accepts any valid CSS properties:

```tsx
<Carousel 
  iconOptions={{
    icon: <span>←</span>,
    iconStyles: {
      // Colors
      color: '#ffffff',
      backgroundColor: '#2563eb',
      
      // Size and typography
      fontSize: '2rem',
      fontWeight: 'bold',
      
      // Border and shadow
      border: '2px solid #1e40af',
      boxShadow: '0 4px 12px rgba(37, 99, 235, 0.3)',
      
      // Effects
      opacity: 0.9,
      transition: 'all 0.3s ease',
      
      // Positioning (overrides defaults)
      width: '50px',
      height: '50px',
    }
  }}
>
  {slides}
</Carousel>
```

## Button Rendering

Icons are rendered inside circular buttons with these default styles (from `CircularButton.tsx:26-40`):

```tsx
{
  borderRadius: "50%",
  width: "45px",
  height: "45px",
  display: "flex",
  alignItems: "center",
  justifyContent: "center",
  boxShadow: "0 2px 8px rgba(0,0,0,0.2)",
  border: "none",
  fontSize: "1.75rem",
  cursor: "pointer",
  // Your iconStyles override these
}
```

<Tip>
  Your `iconStyles` values will override these defaults, giving you full control over the button appearance.
</Tip>

## Theme Integration

Combine `iconOptions` with the `theme` prop for cohesive styling:

```tsx
// Light theme with custom icons
<Carousel 
  theme="light"
  iconOptions={{
    icon: <span>›</span>,
    iconStyles: { 
      color: '#1f2937',
      backgroundColor: '#f9fafb',
      border: '1px solid #e5e7eb'
    }
  }}
>
  {slides}
</Carousel>

// Dark theme with custom icons
<Carousel 
  theme="dark"
  iconOptions={{
    icon: <span>›</span>,
    iconStyles: { 
      color: '#f9fafb',
      backgroundColor: '#1f2937',
      border: '1px solid #374151'
    }
  }}
>
  {slides}
</Carousel>
```

## Icon Visibility

Navigation buttons automatically show/hide based on scroll position:

- **Left button**: Hidden when at the start, visible when scrolled
- **Right button**: Hidden when at the end, visible otherwise

This behavior is handled automatically by the carousel (see `carousel.tsx:65-71`).

## Accessibility

The carousel adds proper ARIA labels to navigation buttons:

```tsx
// Left button
aria-label="Previous item"

// Right button
aria-label="Next item"
```

These labels are built-in and work regardless of your custom icons.
